# アルゴリズム解説

Twigのレイアウトエンジンは、以下の3段階のプロセスでグラフを配置します。

## 1. 階層と親子関係の抽出
まず、GMLデータからノード間の論理的な繋がりを解析します。
- **エッジ消費型探索**: 探索済みのエッジをリストから除外することで、グラフ内の循環参照を回避し、クリーンな木構造を取り出します。
- **双方向参照**: 各ノードは自分の親と子供をリストとして保持し、トラバースを容易にします。

## 2. 描画順序の生成 (BFS)
幅優先探索（Breadth-First Search）を用いて、ノードを階層ごとにグループ化します。
これにより、特定のレベルに存在するノード数に基づいた、動的な幅の割り当てが可能になります。

## 3. 幾何学的な配置計算
各ノードの座標 $$`\(x, y\)`$$とサイズ(width, height)を決定します。
 
### 幅の分配
親ノードが持つ占有幅を、その子ノードの数で均等に分割します。
$$　child\_width = \frac{parent\_width}{num\_of\_children}　$$

### 独自仕様：深さによる高さの拡張
Twigでは、深い階層ほど情報の密度が上がると想定し、以下のルールで高さを設定しています。
- **浅い階層**: 基本サイズで配置。
- **深い階層**: 階層レベルに応じて `height` を拡張し、十分な記述スペースを確保。
